# Local Task Tracking

Use `.tasks/` for lightweight internal task tracking in this repository.

## When to use

- pending follow-up work from the current session
- operational tasks that are not yet implemented
- temporary local tracking when external issue tooling is unavailable

## Format

Create one Markdown file per task.

Recommended fields:

- `Status`
- `Priority`
- `Created`
- `Goal`
- `Context`
- `References`
- `Expected Scope`

## Rules

- keep entries concise and actionable
- link to relevant docs and code paths
- update or remove the task file when the work is completed or superseded
- prefer one task per file rather than a shared running checklist
