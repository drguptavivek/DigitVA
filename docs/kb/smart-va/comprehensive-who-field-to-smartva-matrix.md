# Comprehensive WHO Field To SmartVA Matrix

This document lists every `Id...` field currently configured in DigitVA for `WHO_2022_VA_SOCIAL`, sorted in ascending order.

This matrix is maintained from explicit source trace:

- live ODK form XML saved at `private/odk_form_xml/ICMR01_16_ML01_ICMRVA_WHOVA2022.xml` for WHO field labels
- DigitVA field / form code available in this repo
- `smart-va-pipeline` WHO adapter code under `vendor/smartva-analyze/src/smartva`
- local SmartVA test mapping files bundled in the repo

Where a field id is not present in the current live ODK form XML, the row says so explicitly instead of inferring a label from older docs.

For each field, it shows:

1. the DigitVA WHO field id and label
2. the linked SmartVA WHO intermediate field id or ids when surfaced
3. the linked SmartVA symptom id and label when a tariff-applied symptom path is surfaced
4. a runtime-status note based on the explicit field-by-field SmartVA trace captured in this KB

## Status Meanings

- `explicit mapping surfaced`: a tariff-applied SmartVA path is visible in the current inventory.
- `transformed mapping surfaced`: the field reaches a tariff-applied SmartVA path through recode, duration, multiselect, or helper transformation logic.
- `disconnected-defaulted`: the field is present upstream and a downstream family exists, but the audited runtime path defaults instead of carrying live signal.
- `disconnected-unsurfaced`: a downstream SmartVA family exists, but the visible WHO 2022 bridge is not surfaced in the current adapter.
- `helper-driven`: the path is real, but it depends on helper or prepared fields instead of a single direct question-to-question bridge.
- `no SmartVA mapping surfaced`: no SmartVA path was surfaced for that field in the current inventory.
